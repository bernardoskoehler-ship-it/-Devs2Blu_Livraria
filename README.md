# Devs2Blu_Livraria


## 📖 Sobre o projeto
Este projeto foi desenvolvido como parte de um desafio em grupo do curso +Devs2Blu.

O objetivo da atividade foi analisar um problema real apresentado por uma empresa fictícia e propor a estrutura de um sistema capaz de resolver as dificuldades encontradas, aplicando conceitos de análise de requisitos, planejamento e organização de software.

Embora a proposta original fosse apenas o planejamento da solução, este repositório apresenta a implementação prática do sistema utilizando Java, simulando o funcionamento de uma biblioteca comunitária.

## 📋 Contexto do desafio

A Biblioteca Comunitária Leitura Viva realiza o controle de seus livros e empréstimos por meio de anotações em papel e planilhas.

Com o aumento da quantidade de usuários e livros cadastrados, diversos problemas começaram a surgir:

- Livros emprestados sem controle adequado;

- Dificuldade para identificar livros disponíveis;

- Atrasos na devolução dos empréstimos;

- Cadastros duplicados ou incompletos;

- Perda de tempo na busca por informações;

- Ausência de relatórios para auxiliar a administração da biblioteca.

Nossa missão foi propor um sistema que organizasse esses processos e tornasse o gerenciamento da biblioteca mais eficiente.

## 🎯 Objetivos do sistema

O sistema foi desenvolvido com o propósito de:

- Organizar o cadastro de livros;

- Gerenciar usuários da biblioteca;

- Controlar empréstimos e devoluções;

- Evitar inconsistências nos registros;

- Facilitar consultas rápidas;

- Automatizar processos que antes eram realizados manualmente;

- Disponibilizar informações úteis para a administração da biblioteca.

## ⚙️ Funcionalidades

O sistema contempla funcionalidades como:

- Cadastro de livros;
- Cadastro de usuários;
- Realização de empréstimos;
- Registro de devoluções;
- Consulta de livros cadastrados;
- Consulta de usuários;
- Controle de disponibilidade dos livros;
- Relatórios básicos do sistema.

## 🚀 Tecnologias utilizadas

- Java
- Programação Orientada a Objetos (POO)
- Estruturas de repetição
- Estruturas condicionais
- Coleções (List, ArrayList)
- Entrada de dados com Scanner

Este projeto tem finalidade educacional e foi desenvolvido para praticar:

- Levantamento de requisitos;
- Planejamento de software;
- Modelagem de um sistema;
- Organização de código em Java;
- Trabalho em equipe;
- Desenvolvimento de aplicações orientadas a objetos.
